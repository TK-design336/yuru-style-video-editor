# ⓪ マルチトラック同期 — 設計仕様（予定）

実務向けの **ピンマイク等・複数マイク撮影** に対応するため、① 文字起こしの前段に位置する前処理ステップを追加する。

---

## 背景・目的

対談・インタビュー撮影では、カメラ内蔵マイクとは別に話者ごとのピンマイクやレコーダーで音声を収録することが多い。映像と各音声トラックは **録画開始タイミング・機器のクロック差** によりオフセットや尺のずれが生じる。

本ステップでは、長さや開始時刻の異なる動画・音声トラックを **動画タイムライン基準に揃え**、以降の文字起こし・話者分離の入力品質を確保する。

---

## 入力モード

`meta.audio_sources` で取り込み形態を区別する（詳細は [DATAMODEL.md](./DATAMODEL.md)）。

| モード | 入力 | 本ステップ |
|---|---|---|
| **混合音声** | 動画 1 本（内蔵マイク等の混合トラックを含む） | **スキップ** → 既存の ① 文字起こしへ |
| **話者別トラック** | 動画 1 本 + 話者別音声ファイル N 本 | **本ステップを実行** → 同期確認後 ① へ |

話者別トラックモードの入力例:

- 映像: `camera.mp4`
- 音声: `host_mic.wav`（話者1）, `guest_mic.wav`（話者2）, `room_ambient.wav`（その他）

---

## 処理フロー（全体）

```
動画 + 話者別音声 N 本
        ↓
⓪ マルチトラック同期（本ドキュメント）
   ・各トラックのオフセット算出
   ・confidence 低いトラックは人手で微調整
        ↓
① 文字起こし
   ・文字起こし用音声 = 話者トラックを同期・統合したミックス
   ・話者分離 = 各時刻で RMS エネルギー最大のトラックを話者と判定
        ↓
② 以降（既存ワークフロー）
```

### ① 文字起こしとの連携方針

| 処理 | 方針 |
|---|---|
| **文字起こし** | 話者用音声データを **同期オフセット適用後に統合** したミックスから Whisper で実行 |
| **話者分離** | NeMo MSDD 等のクラスタリングに頼らず、**各フレーム／区間で RMS エネルギーが最大の話者トラック** を話者 ID として割り当て（ピンマイク収録では物理的に分離されているため確実） |
| **混合音声モード** | 従来どおり動画内音声のみで文字起こし + MSDD 話者分離 |

## オフセット算出（`python/sync_media.py` — 予定）

### 入力（stdin JSON）

```jsonc
{
  "video_path": "/path/to/camera.mp4",
  "speaker_tracks": [
    { "speaker_id": "SPK_0", "path": "/path/to/host_mic.wav" },
    { "speaker_id": "SPK_1", "path": "/path/to/guest_mic.wav" }
  ]
}
```

### 出力（stdout JSON）

```jsonc
{
  "ok": true,
  "data": {
    "offsets_ms": { "SPK_0": 120, "SPK_1": -45 },
    "confidence": { "SPK_0": 0.92, "SPK_1": 0.41 }
  }
}
```

`offsets_ms[speaker_id]` は **話者トラックを動画タイムラインに合わせるためのシフト量（ミリ秒）**。正の値はトラックを遅らせる（後ろにずらす）方向とする。

### Tier 1 — FFT クロスコリレーション（動画に音声がある場合）

1. **ffmpeg** で動画から音声を抽出（モノラル PCM）
2. 各話者音声ファイルと **FFT ベースの相互相関** でオフセット（ms）を算出
   - `numpy.fft.fft` / `numpy.fft.ifft` による高速相関
   - **2 段階方式**:
     - 粗探索: 8 kHz 程度にダウンサンプルした信号で相関ピークを検出
     - 精密化: 見つかったオフセット周辺のみ元サンプルレートで再計算
3. 各トラックについて **相関ピーク高さ / ノイズフロア比** から `confidence`（0〜1）を算出

### Tier 3 — メタデータフォールバック（動画に音声がない場合）

動画トラックに音声ストリームが存在しないとき:

1. **ffprobe** で各ファイルの `creation_time`（または同等のタイムスタンプメタデータ）を読み取り
2. 動画を基準とした差分を **粗いオフセット初期値** として返す
3. `confidence` は常に低めの固定値（**約 0.3**）— 人手確認を前提とする

## UI（`SyncPanel.tsx` — 予定）

`Ingest.tsx` に統合。動画 + 話者別音声が入力された場合のみ表示する。

| 要素 | 仕様 |
|---|---|
| 波形オーバーレイ | wavesurfer.js の複数トラック表示を流用。動画基準タイムライン上に各話者トラックを重ね表示 |
| 警告バッジ | `confidence < 0.6` のトラックに表示 |
| 手動微調整 | **警告のあるトラックのみ** ±ms スライダー + 数値入力（波形を目視しながら調整） |
| 自動適用 | `confidence >= 0.6` のトラックは UI を出さず算出オフセットをそのまま適用 |

同期確定後、`meta.audio_sources` にオフセット・confidence を書き込み、① 文字起こしへ進む。

```jsonc
  "mode": "per_speaker",
  "tracks": [
    {
      "speaker_id": "SPK_0",
      "path": "/path/to/host_mic.wav",
      "offset_ms": 120,
      "sync_confidence": 0.92
    }
  ],
  "sync_completed_at": "2026-06-18T12:00:00Z"
}
```
